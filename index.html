Thermal Rat Detection

Thermal Rat Detection for Farmers

Overview

A real-time nocturnal rat detection system combining thermal camera imagery and YOLOv11 object detection, deployed to solve an actual pest problem identified during fieldwork in Desa Sengka. Detection triggers an automatic ultrasonic deterrent.

Problem

Farmers often face:

  • Night-time rat attacks that are difficult to detect
  • Ineffective traditional monitoring methods
  • Limited access to affordable automated detection systems

Solution

We collected thermal rat datasets from Roboflow and Kaggle and developed a computer vision project that:

  • Collected and unified thermal rat datasets into YOLO format.
  • Applied thermal-specific data augmentation to improve detection robustness
  • Integrated real-time rat detection with automatic ultrasonic activation.

My Role

ML Developer & Lead Author

  • Collected thermal datasets and converted segmentation annotations into YOLO bounding-box format
  • Developed a thermal-image augmentation pipeline to improve model robustness and detection accuracy
  • Trained and evaluated the YOLOv11-based thermal rat detection system for real-time inference
  • Integrated automated ultrasonic activation triggered by detection events
  • Contributed to academic research writing and project documentation

Tech Stack

Training Environment Google Colab GPU
Dataset Sources Kaggle & Roboflow
Detection Model YOLOv11

Challenges

  • Thermal images lacked RGB texture and color information, requiring specialized augmentation techniques.
  • The Kaggle dataset used segmentation masks, requiring custom conversion into YOLO bounding-box annotations.
  • No thermal camera hardware was available, so development relied entirely on open-source datasets and YouTube thermal videos.
  • Model evaluation used unseen real-world thermal videos to better test generalization performance.

Impact

The system achieved mAP@50 above 0.99, precision 0.989, and recall 0.97+, performance strong enough to operate reliably as an early-warning system. During inference testing, the model consistently detected rats in external thermal video and successfully triggered the ultrasonic emitter module.

What I'd Improve Next

  • Deploy on a Raspberry Pi or Jetson Nano with an actual thermal camera for in-field validation
  • Extend detection to multiple pest species (snakes, insects) by expanding the dataset
  • Build a mobile alert system so farmers receive real-time notifications on their phones

Order Here – Smart Food Ordering Ecosystem

Overview

Order Here is a smart food ordering ecosystem developed for Universitas Ciputra Makassar to modernize the campus dining experience through artificial intelligence, IoT, and web technologies.

Instead of creating a standalone recommendation model, our team designed a complete ecosystem consisting of a web-based ordering platform, MoodBites mobile application, IoT-powered NFC authentication kiosk, cloud backend, and an AI recommendation microservice.

My primary contribution focused on designing and developing the machine learning recommendation engine that generates personalized food recommendations based on users' taste preferences, favorite foods, and weighted flavor priorities.

Problem

Campus cafeterias typically provide static menus with no personalization, making it difficult for students to discover meals that match their preferences while maintaining a smooth ordering experience.

  • Food recommendations were generic rather than personalized.
  • First-time users had no recommendation history.
  • Ordering process required manual identification and payment.
  • Existing systems lacked integration between ordering, authentication, and recommendation services.

Solution

We built a complete smart food ordering ecosystem consisting of multiple interconnected services.

  • AI-powered food recommendation engine using a custom Weighted Euclidean KNN algorithm.
  • FastAPI recommendation microservice deployed independently using Docker.
  • Web-based ordering platform for campus cafeteria operations.
  • MoodBites mobile application allowing users to submit their taste preferences.
  • IoT kiosk utilizing Raspberry Pi and PN532 NFC reader for student authentication.
  • Cloud deployment using VPS and Supabase for centralized data management.

System Architecture

The recommendation engine was designed as an independent microservice integrated with the larger Order Here ecosystem.

Recommendation Pipeline

The recommendation engine follows a content-based recommendation approach, utilizing a custom Weighted Euclidean K-Nearest Neighbors algorithm to compute similarity scores between user preferences and available menu items.

Key Features

  • Personalized food recommendations based on user taste preferences.
  • Weighted Euclidean KNN algorithm for similarity calculation.
  • Favorite-food bonus scoring to improve recommendation relevance.
  • Category-aware recommendation generation for balanced meal suggestions.
  • RESTful API integration with the Order Here ecosystem.
  • Dockerized deployment for scalable microservice architecture.

Challenges

  • Designing a recommendation algorithm that accurately reflects users' taste priorities rather than treating every preference equally.
  • Building a recommendation service capable of communicating seamlessly with web, mobile, and IoT components.
  • Ensuring low response latency for real-time recommendation requests.
  • Balancing recommendation diversity while maintaining personalization quality.
  • Coordinating API contracts across multiple development teams.

Results

  • Successfully implemented a personalized food recommendation engine using a custom Weighted Euclidean KNN algorithm.
  • Integrated the recommendation service into a complete smart food ordering ecosystem.

  • Containerized the recommendation service using Docker for easier deployment and maintenance.

Impact

COPPA Risk Prediction

Overview

A machine learning project developed for the FIND IT 2025 Data Analytics Competition, aiming to predict whether mobile applications violate the Children's Online Privacy Protection Act (COPPA). The project leverages TPOT AutoML to automatically discover the best preprocessing and classification pipeline.

Problem

Identifying applications with potential COPPA violations requires analyzing numerous metadata features and selecting an effective machine learning pipeline.

  • Large number of heterogeneous app features.
  • Manual model selection is time-consuming.
  • Feature engineering significantly affects prediction quality.

Solution

  • Engineered 17 predictive features, including child-friendly rating indicators.
  • Applied TPOT AutoML to automatically search preprocessing and classifier combinations.
  • Compared multiple generated pipelines to identify the most effective model.
